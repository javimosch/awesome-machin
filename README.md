# Awesome machin [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

A curated list of things built with **[machin](https://github.com/javimosch/machin)** ‚Äî MFL, the Machine-First Language. machin compiles a terse, type-inferred, agent-friendly language to native code through C: small static binaries, no runtime.

> New machin app? Open a PR adding a one-line entry. Keep the list machine-first: terse, linked, current.

## The language

- **[machin](https://github.com/javimosch/machin)** ‚Äî the compiler + language (MFL ‚Üí C ‚Üí native). Generics, closures, goroutines/channels (`select`, `close`, range, comma-ok), arena GC, C FFI, a `machweb` web framework, and a growing builtin surface (networking, native TLS + WebSocket, file I/O, SQLite, JSON + jq-style path queries, regex, base64, SHA-256/HMAC, `(value, err)` error handling).

> Organized by domain: **CLI tools & services**, **Game dev**, **Web**, **Crypto & protocols**. New repos follow a domain-scoped naming convention ‚Äî `machin-game-demo-*` for games/graphics, `machin-web-demo-*` for web.

## Learning & reference

- **[machin-learn](https://github.com/javimosch/machin-learn)** ‚Äî a **knowledge base for agents** learning MFL: seven skill files covering the language, builtins, gotchas/pitfalls, terminal/TUI, networking, storage/SQLite, and C FFI. Each skill is a self-contained `.md` an agent can load in one `read` call, with concrete examples and warnings for every caveat ‚Äî designed to help budget models master machin fast.

## CLI tools & services

- **[boilerplate-cli-ui-machin](https://github.com/javimosch/boilerplate-cli-ui-machin)** ‚Äî a single-binary CLI serving an embedded React web UI + JSON API, with a background daemon via the C FFI. The machin entry in the [SuperCLI](https://github.com/javimosch) boilerplate family (smallest binary at ~27 KB).
- **[machin-blob](https://github.com/javimosch/machin-blob)** ‚Äî a binary **blob inspector & patcher**: reads typed fields (`i32`, `f32`, `str`) at exact byte offsets via `peek_i32`/`peek_f32`, patches hex values at offsets, slices byte ranges. Drove **low-level memory ops** (`alloc`, `free`, `poke_u8`, `peek_i32`, `peek_f32`, `poke_i32`/`poke_f32`, `poke_ptr`) into the CLI-tool domain.
- **[machin-healthcheck](https://github.com/javimosch/machin-healthcheck)** ‚Äî a concurrent HTTP status/latency checker. One goroutine per URL, results over a channel. Drove `dial`/`now_ms`/`parse_int` into machin.
- **[machin-gen](https://github.com/javimosch/machin-gen)** ‚Äî a **template engine**: replaces `{{key}}` placeholders with values, supports transform pipelines (`{{name|upper|reverse}}`) via MFL's first-class **closures** stored in a `map[string]func` dispatch table. Transforms: upper, lower, reverse, trim, len, hex. Drove **closures** (`func` values, `[]func`, `map[string]func`) and **piped transforms** into the ecosystem.
- **[machin-hexdump](https://github.com/javimosch/machin-hexdump)** ‚Äî a binary **hex viewer**: classic hexdump format (offset / hex / ASCII sidebar), configurable width, colored output. Drove **`bytes`** builtins (`read_file_bytes`, `byte_at`, `to_hex`, `bytes_str`) and the `byte_to_hex` pattern into the CLI-tool domain.
- **[machin-ssg](https://github.com/javimosch/machin-ssg)** ‚Äî a static-site generator (markdown subset ‚Üí HTML). Drove native file I/O and a parser fix into machin.
- **[machin-fetch](https://github.com/javimosch/machin-fetch)** ‚Äî a tiny HTTPS client CLI (a pocket `curl` for `https://`). GET/POST, single binary. Drove **native TLS** (`https_get`/`https_post`, OpenSSL) into machin.
- **[machin-wscat](https://github.com/javimosch/machin-wscat)** ‚Äî a tiny duplex WebSocket (`wss://`) client CLI (a pocket `websocat`). Stdin ‚Üí send, recv ‚Üí stdout. Drove a **native WebSocket client** (`wss_open`/`send`/`recv`/`close`, RFC 6455 over TLS) into machin.
- **[machin-linkcheck](https://github.com/javimosch/machin-linkcheck)** ‚Äî a concurrent link checker that classifies each URL (OK / 4xx / 5xx / unreachable) and exits non-zero on broken links. Drove **error handling** into machin: `http_get` returning `(status, body, err)` plus `exit`.
- **[machin-jq](https://github.com/javimosch/machin-jq)** ‚Äî a tiny `jq`-style JSON path query CLI (`.a.b[0].name`). Pairs with machin-fetch for a native HTTPS‚Üíquery pipeline. Drove a **JSON path engine** (`json_get`) into machin.
- **[machin-pool](https://github.com/javimosch/machin-pool)** ‚Äî a bounded-concurrency URL checker: a fixed worker pool racing results against an overall deadline. Drove **`select`** (multi-way channel waits, timeouts) into machin.
- **[machin-pipe](https://github.com/javimosch/machin-pipe)** ‚Äî a streaming parallel fetch pipeline: producer ‚Üí workers ‚Üí fan-in, each stage terminating by closing its channel. Drove **channel `close`** + range-over-channel into machin.
- **[machin-batch](https://github.com/javimosch/machin-batch)** ‚Äî groups a line stream into JSON-array chunks by size or by timer (flushing the rest on EOF). Its `select` over input + ticker drove the **comma-ok receive** (`v, ok := <-ch`) into machin.
- **[machin-http](https://github.com/javimosch/machin-http)** ‚Äî a multi-command HTTPS client (`get`/`post`/`head`) with short+long flags and auto `--help`. Drove a reusable **flag parser** (`framework/flags.src`) into machin.
- **[machin-grep](https://github.com/javimosch/machin-grep)** ‚Äî a regex line matcher (pocket `grep`): match/invert/count/only/line-numbers over a POSIX ERE. Drove **regex** builtins (`regex_match`/`find`/`groups`/`replace`) into machin.
- **[machin-jwt](https://github.com/javimosch/machin-jwt)** ‚Äî a JWT decoder: decode header/payload (base64url JSON) and extract a claim. Drove **base64** builtins (`base64_encode`/`base64_decode`) into machin.
- **[machin-hmac](https://github.com/javimosch/machin-hmac)** ‚Äî compute/verify HMAC-SHA256 (or SHA-256) signatures ‚Äî webhook verification (GitHub/Stripe). Drove **hash** builtins (`sha256`/`hmac_sha256`) into machin.
- **[machin-wc](https://github.com/javimosch/machin-wc)** ‚Äî a `wc` clone (lines/words/bytes), matching coreutils. Drove **`read_stdin`** (slurp stdin verbatim) into machin.
- **[machin-kv](https://github.com/javimosch/machin-kv)** ‚Äî a persistent key-value store (set/get/del/list), data surviving across runs. Drove **SQLite** builtins (`sqlite_open`/`exec`/`query`/`close`) into machin.
- **[machin-cron](https://github.com/javimosch/machin-cron)** ‚Äî a cron-expression evaluator: prints the next N times a crontab line fires. Drove **`time_fields`** (decompose a Unix timestamp into calendar fields) into machin.
- **[machin-date](https://github.com/javimosch/machin-date)** ‚Äî a `date(1)`-style timestamp formatter (output matches GNU `date`). Drove **`time_format`** (strftime over a Unix timestamp) into machin.
- **[machin-mcp](https://github.com/javimosch/machin-mcp)** ‚Äî a **Model Context Protocol (MCP) server**: JSON-RPC 2.0 over stdio transport, exposing 6 tools (echo, now, read_file, write_file, list_dir, sqlite_query). The smallest MCP server in the ecosystem ‚Äî 31 KB static binary, no Node/Python runtime. Composes `json_get`/`json()` for JSON-RPC, `input()`/`println()`/`flush()` for stdio, `sqlite_open`/`sqlite_query` for databases, `read_file`/`write_file`/`list_dir` for filesystem ‚Äî no new builtins.
- **[machin-meet](https://github.com/javimosch/machin-meet)** ‚Äî a minimal self-hostable **Calendly for one person**: a single binary (HTTP + SQLite + embedded booking page) serving free slots, `.ics` files, and signed manage/cancel links, with booking alerts via **machin-notify** (Discord/Telegram) or WhatsApp. Drove **`time_make`** (build a Unix timestamp from calendar fields) into machin, completing the time trio.
- **[machin-qs](https://github.com/javimosch/machin-qs)** ‚Äî a URL query-string ‚áÑ JSON converter (decode to a JSON object, or build a percent-encoded query from `key=value` pairs). Drove **`url_encode`/`url_decode`** (RFC 3986 percent-encoding) into machin.
- **[machin-notify](https://github.com/javimosch/machin-notify)** ‚Äî a notification **hub** (daemon + CLI): configure Discord webhooks / Telegram bots once, then apps `POST /notify {token, channel, message}` to one local endpoint ‚Äî no provider integration or secrets in each app. machin-meet sends booking alerts through it.
- **[machin-db-migrate](https://github.com/javimosch/machin-db-migrate)** ‚Äî a **SQLite ‚áÑ MongoDB** migration CLI, both directions, in one pure-MFL binary (no driver, no `mongoexport`). Drove a pure-MFL **MongoDB client** into machin ‚Äî the **OP_MSG wire protocol** + a **BSON codec** ([`framework/mongo.src`](https://github.com/javimosch/machin/blob/main/framework/mongo.src) + [`bson.src`](https://github.com/javimosch/machin/blob/main/framework/bson.src)). `sqlite2mongo` reads the table's `PRAGMA` schema ‚Üí typed BSON docs; `mongo2sqlite` infers columns from the first document ‚Üí parameterized inserts; verified by a `SQLite ‚Üí Mongo ‚Üí SQLite` round-trip.
- **[machin-watch](https://github.com/javimosch/machin-watch)** ‚Äî a tiny **uptime monitor**: polls URLs on a per-watch interval and alerts on every up‚Üîdown transition (with recovery time) through **machin-notify**. Composes `http_get` + SQLite + `sleep` ‚Äî no new builtins; closes the loop (deploy ‚Üí notify ‚Üí monitor).
- **[machin-pomo](https://github.com/javimosch/machin-pomo)** ‚Äî a terminal **Pomodoro timer**: live ANSI rendering, keyboard controls (pause/resume/skip/quit), configurable work/break durations, automatic long-break scheduling. Drove **`raw_mode`/`read_key`** (cbreak/no-echo + non-blocking single-key reads) into the CLI-tool domain: the terminal-input layer every future TUI inherits.
- **[machin-qr](https://github.com/javimosch/machin-qr)** ‚Äî a terminal **QR code generator**: encodes text into a scannable QR code via MFL's C **FFI** (`libqrencode`), rendered with ANSI background colors. Drove **`extern`** (FFI declarations), **raw memory access** (`peek_i32` for struct fields), and **pointer reconstruction** (64-bit little-endian from two i32 halves) into the ecosystem.
- **[machin-revproxy](https://github.com/javimosch/machin-revproxy)** ‚Äî a **reverse proxy**: route HTTP requests to backend servers by path prefix, with header rewriting (Host, X-Forwarded-*) and prefix stripping. Longest-prefix match routing, concurrent goroutines per request. Composes `dial`/`read`/`write` for TCP relay, `read_bytes` for binary-safe response forwarding, `has_prefix` for route matching ‚Äî no new builtins.
- **[machin-scan](https://github.com/javimosch/machin-scan)** ‚Äî a concurrent **port scanner**: goroutine worker pool, bounded concurrency, channel-based result collection via `select`, optional timeout. Scans port ranges, comma-separated lists, or single ports with service name lookup (30+ common ports). Drove **channels** (`make(chan T)`/`<-ch`/`close(ch)`) and **`select`** (multi-way channel wait) into the CLI-tool domain ‚Äî MFL's full concurrency model.
- **[machin-serve](https://github.com/javimosch/machin-serve)** ‚Äî a static file **HTTP server**: concurrent (goroutines per request), opt-in Basic auth, MIME-type detection, directory index (`index.html`), path-traversal protection. Binds to `0.0.0.0` with configurable port and directory. Composes `listen`/`accept`/`read`/`write`/`close` + `read_file` + `regex_groups` + `base64_decode` + `go` ‚Äî no new builtins.
- **[machin-top](https://github.com/javimosch/machin-top)** ‚Äî a live **system dashboard**: real-time CPU, memory, uptime, and load average with ANSI-colored progress bars. Reads `/proc/stat`, `/proc/meminfo`, `/proc/uptime`, `/proc/loadavg` via `read_file`. Refreshes live with `sleep` and ANSI cursor control from pomo ‚Äî no new builtins.

- **[machin-game-roguelike](https://github.com/javimosch/machin-game-roguelike)** ‚Äî a full **terminal roguelike** with **native emoji rendering**: procedural dungeon generation (BSP rooms + corridors), field-of-view via raycasting, monster chase AI, turn-based combat, RPG leveling, inventory, and **JSON save/load** ‚Äî all in pure MFL, 49 KB. Exercises `rand_bytes` (RNG), `sin`/`cos`/`floor` (FOV), `json`/`parse` (persistence), and multi-system game state orchestration. The deepest single-player game in the ecosystem.
- **[machin-game-demo-snake](https://github.com/javimosch/machin-game-demo-snake)** ‚Äî the classic **Snake** as a native terminal game in pure MFL: real-time WASD/HJKL movement, a growing snake, ANSI rendering, a frame loop. Drove **`raw_mode`/`read_key`** (cbreak/no-echo terminal mode + non-blocking single-key reads) into machin ‚Äî the terminal-input layer every future TUI inherits.
- **[machin-game-demo-2048](https://github.com/javimosch/machin-game-demo-2048)** ‚Äî the **2048** sliding-puzzle as a real **GUI desktop app**: a native OpenGL window drawn with [raylib](https://www.raylib.com/) through machin's **C FFI** (scalars + a by-value `Color` struct). Pure-MFL board logic (one slide routine for all four directions); the GUI counterpart to machin-game-demo-snake. Composes the FFI ‚Äî no new builtins.
- **[machin-game-demo-flappy](https://github.com/javimosch/machin-game-demo-flappy)** ‚Äî a **Flappy-Bird** with real PNG **sprites**: `Texture2D` handles returned across the FFI, `f32`-field `Rectangle`/`Vector2` structs for `DrawTextureRec`/`DrawTexturePro`, a sprite sheet rotated by velocity, pipes flipped via a negative source height. The texture/sprite FFI dogfood ‚Äî and its physics drove **`float()`** (int‚Üífloat conversion) into machin.
- **[machin-game-demo-simon](https://github.com/javimosch/machin-game-demo-simon)** ‚Äî **Simon Says** with **sound**: flash-and-tone a growing sequence, repeat it by clicking. The audio dogfood ‚Äî raylib's pointer-bearing `Sound` struct drove **FFI opaque handles** (`cstruct Sound {}`) into machin, so a by-value C handle can be loaded, stored in `[]Sound`, and passed back to `PlaySound` without naming its fields.
- **[machin-game-demo-3d](https://github.com/javimosch/machin-game-demo-3d)** ‚Äî a real-time **3D** scene: a camera orbits a ring of bobbing cubes on a grid. The 3D dogfood ‚Äî raylib's `Camera3D` is a by-value struct of three `Vector3`s, which drove **FFI nested cstructs** (a `cstruct` field that is another `cstruct`, marshaled recursively) into machin, so `BeginMode3D(Camera3D{Vector3{...}, ...})` works. Its libm camera orbit then drove **native math** builtins into machin.
- **[machin-game-demo-anim](https://github.com/javimosch/machin-game-demo-anim)** ‚Äî a **2D procedural animation**: 1200 particles swirl through a flow field around three drifting vortex centers, every pixel of motion `sin`/`cos`/`atan2`/`hypot`/`pi` over time, hue by flow direction (`ColorFromHSV`). The showcase for machin's **native math** builtins ‚Äî pure composition, no new feature.
- **[machin-game-demo-terrain](https://github.com/javimosch/machin-game-demo-terrain)** ‚Äî **procedural mesh generation**: an animated terrain whose every vertex is computed in MFL from layered trig, flat-shaded by its face normal, and streamed to the GPU triangle-by-triangle with raylib's immediate-mode rlgl (`rlBegin`/`rlVertex3f`) via a headerless extern. Composition again ‚Äî and it points at the **pointer/array FFI** gap for real GPU vertex buffers. See the [game-dev north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-GAMEDEV.md).
- **[machin-game-demo-planet](https://github.com/javimosch/machin-game-demo-planet)** ‚Äî a procedural **GPU mesh**: a curved planet-surface chunk built **once** in raw memory (vertex/color arrays + the raylib `Mesh` struct), uploaded to a GPU vertex buffer, and drawn as a model. Drove machin's **pointer/array FFI** (v0.47.0) ‚Äî `alloc`/`poke_*` raw buffers and the `*Mesh` deref param (`UploadMesh`/`LoadModelFromMesh`). The first time machin hands C a raw pointer/array; the Tier-2 unlock toward planet-scale meshes.
- **[machin-game-demo-cyberpunk](https://github.com/javimosch/machin-game-demo-cyberpunk)** ‚Äî an **infinite procedural planet you fly through**: endless `noise2`/fbm wasteland terrain streamed in GPU-mesh chunks around a WASD+mouse fly camera, grimy Blade-Runner megastructures (100+ variants) clustered in city districts, **GPU-instanced flora** (thousands of plants in one `DrawMeshInstanced` call) in the scrubland between, a herd of **procedurally-animated skeletal creatures** (forward-kinematics walkers posed through the rlgl matrix stack, sine-driven diagonal gait), drifting drones, **real depth fog via a post-processing shader**, and a **10 km draw distance** (custom projection matrix + a coarse recentered terrain underlay). Drove machin's **`noise` builtins** (v0.49.0); everything else ‚Äî buildings, instancing, shaders, fog, skeleton animation, far-clip override ‚Äî is composition. The Tier-3 "procedural worlds" step, and proof the FFI reaches raylib's shaders, instancing, matrix stack, and custom projection with no new language feature.
## Web

> machin does both ends of the web in one language ‚Äî a native server (machweb) and a reactive WebAssembly UI ‚Äî no Node/bundler. The agent how-to is the [`machin-web` skill](https://github.com/javimosch/machin/blob/main/skills/machin-web/SKILL.md); the direction is the [web north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-WEB.md). (Web *services* like [machin-meet](https://github.com/javimosch/machin-meet) and [machin-ssg](https://github.com/javimosch/machin-ssg) are listed under CLI tools & services.)

- **[machin-saas-demo](https://github.com/javimosch/machin-saas-demo)** ‚Äî **MachNotes**, an end-to-end **SaaS in one binary**: the whole backend stack wired together ‚Äî **OAuth2/OIDC SSO** login ([`sso.src`](https://github.com/javimosch/machin/blob/main/framework/sso.src), with a bundled mock identity provider so it runs locally), **sessions in Redis** (session id ‚Üí email, TTL; [`redis.src`](https://github.com/javimosch/machin/blob/main/framework/redis.src)), a **parameterized Postgres** notes table ([`postgres.src`](https://github.com/javimosch/machin/blob/main/framework/postgres.src)), signed cookies + routing ([`machweb.src`](https://github.com/javimosch/machin/blob/main/framework/machweb.src)), and a **reactive WebAssembly UI** ([`reactive.src`](https://github.com/javimosch/machin/blob/main/framework/reactive.src) ‚Üí wasm, keyed-list patches). Login ‚Üí store note ‚Üí reactive list ‚Üí logout, verified end to end. The backend capstone ‚Äî proof a single static MFL binary is a credible SME backend (HTTP + auth + SQL + cache), no Node/cgo/client-libs. Concurrency-safe via connection pooling (the gap it surfaced, added in machin v0.65.0). See the [backend north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-BACKEND.md).
- **[boilerplate-cli-ui-machin-isomorphic](https://github.com/javimosch/boilerplate-cli-ui-machin-isomorphic)** ‚Äî a **starting point for isomorphic web apps in machin**: one static native binary that is a CLI, an HTTP server, a JSON API, and serves its own **reactive WebAssembly UI** ‚Äî the whole app (server, client, shared model) in MFL, no Node/bundler. The example poll is **server-rendered** (works with JS off), then **hydrated** by a wasm client where signals + a computed total + templating drive surgical DOM updates; `models.src` is compiled into *both* ends. Composes everything: `machweb` + `flags` + `reactive` + `--target wasm` + binary-body serving. The full-stack-MFL capstone.
- **[machin-web-demo-router](https://github.com/javimosch/machin-web-demo-router)** ‚Äî a **multi-page SPA** with client-side routing: three pages, no-reload navigation, working back/forward + address bar, and deep-links (refresh on `/users`). The router ([`framework/router.src`](https://github.com/javimosch/machin/tree/main/framework), on top of reactive) makes the active route a **signal** (an int index); an `outlet` reaction re-renders the active page, `link`/`navigate` switch routes, and path strings flow host‚Üíwasm via `ptr_str`. One page is reactive ‚Äî a counter whose state **persists across navigation** (it's a signal). A catch-all machweb server serves the shell for any path. All MFL.
- **[machin-web-demo-users](https://github.com/javimosch/machin-web-demo-users)** ‚Äî a **CRUD back-office** (manage a users database): one native binary that is a CLI + HTTP server + **SQLite**-backed JSON API + reactive wasm UI (list / add / toggle active / delete). The server owns the data (`sqlite_query` ‚Üí JSON rows, `?`-bound params); the client is a reactive *view* ‚Äî the host fetches `/api/users` and hands the JSON to the wasm (`ptr_str` + `json_get`), a keyed list patches only changed rows, a computed shows the active count, and a form adds users. The SPA-over-API counterpart to the isomorphic boilerplate; the worked "manage my users DB" recipe.
- **[machin-web-demo-todo](https://github.com/javimosch/machin-web-demo-todo)** ‚Äî a reactive **todo app with real text input**: state, logic, and rendering all in machin, compiled to wasm. The new piece is a working `<input>` ‚Äî the typed UTF-8 flows *into* the module (machin v0.57's **`ptr_str`**: the host writes the text into wasm memory at an `alloc`'d pointer, machin reads it). Signals + a computed "active" count + a **keyed list** (re-keyed by `id*100+done` so a toggle re-renders just that row) + templating. Adding inserts one row, toggling re-renders one, deleting removes one ‚Äî no `innerHTML` churn. Rounds out the toolkit with **forms**.
- **[machin-web-demo-reactive](https://github.com/javimosch/machin-web-demo-reactive)** ‚Äî **fine-grained reactivity** (the Solid/Leptos model, in MFL): a reactive list whose **state and view logic live entirely in machin** ‚Äî **signals**, a **computed** sum, live bindings, and a **keyed list** that emits only the DOM deltas (`insert`/`remove`/`reorder`), no `innerHTML` replacement, no virtual-DOM diff. The runtime ([`framework/reactive.src`](https://github.com/javimosch/machin/tree/main/framework)) auto-tracks which reactions read which signals, recomputes only the affected ones, and patches only when the rendered text/keys actually change ‚Äî sorting the list emits *only* a reorder, no item re-render. Drove **`[]func`** (slices of closures) into machin (v0.53.0) and **computed + keyed reconciliation** (v0.54.0); composes package globals (the signal store) + `--target wasm`.
- **[machin-web-demo-ssr](https://github.com/javimosch/machin-web-demo-ssr)** ‚Äî **isomorphic server-side rendering**: a counter page rendered by a single native machin binary (machweb), where the view logic lives in a shared `view.src` that *also* compiles to the wasm client ‚Äî **one component model, two runtimes**. The server's HTML and the wasm client's re-render are byte-for-byte identical (same `view()`). SSR needs **no JavaScript** (the +/‚àí controls are plain server-round-trip links); the wasm client adds an instant SPA from the same source. No Node, no bundler ‚Äî `machin encode` + `machin build`. The full-stack-MFL direction: write a component once, run it on the server for first paint and in wasm for interactivity.
- **[machin-web-demo-wasm](https://github.com/javimosch/machin-web-demo-wasm)** ‚Äî **machin on the frontend**: an interactive counter whose entire view (markup, `n¬≤`, a recursive `fib`) is written in MFL, compiled to **WebAssembly**, and rendered live in the browser ‚Äî the way Rust does with Yew/Leptos, here **MFL ‚Üí C ‚Üí wasm** reusing machin's C backend. `zig cc` (clang + wasi-libc, one binary) cross-compiles the emitted C to a `wasm32-wasi` reactor module with zero WASI imports, so it loads in a bare page. The DOM bridge **is machin's own FFI, both ways**: exported MFL functions are wasm exports JS calls into; an `extern "env"` block becomes a wasm import the host supplies (machin ‚Üí `innerHTML`). Composition ‚Äî machin reaches the browser with no new language primitive, only a leaner emit. The first **frontend** dogfood; see the [web north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-WEB.md).
## Crypto & protocols

> The pure-MFL layers behind the native WhatsApp client stack (see the case study below).

- **[machin-protobuf](https://github.com/javimosch/machin-protobuf)** ‚Äî a protobuf wire-format inspector (`protoc --decode_raw`-style) + reusable codec library, implemented in pure MFL over the **`bytes`** type. Exercises machin's binary surface (bytes + crypto + binary WebSocket) built out for a native WhatsApp client.
- **[machin-noise](https://github.com/javimosch/machin-noise)** ‚Äî a [Noise Protocol](https://noiseprotocol.org) **XX handshake** (`Noise_XX_25519_AESGCM_SHA256`, the pattern WhatsApp uses) in pure MFL over the `bytes` + crypto builtins, with a loopback self-test. The transport-encryption layer for a native WhatsApp client.
- **[machin-wabin](https://github.com/javimosch/machin-wabin)** ‚Äî the **WhatsApp binary-node (stanza) codec** in pure MFL: token-dictionary compression (from [whatsmeow](https://github.com/tulir/whatsmeow)), nibble/hex packing, the node tree. Encoder output is byte-for-byte WhatsApp-compatible. The stanza-framing layer for a native WhatsApp client.
- **[machin-signal](https://github.com/javimosch/machin-signal)** ‚Äî the **Signal Double Ratchet** (X3DH + ratcheting cipher) in pure MFL, key schedule matching [libsignal](https://github.com/tulir/libsignal-protocol-go) exactly, with a loopback self-test. The message-encryption layer for a native WhatsApp client.
- **[machin-wapair](https://github.com/javimosch/machin-wapair)** ‚Äî WhatsApp **device-pairing crypto** (the ADV account/device signature exchange) in machin, modeling [whatsmeow](https://github.com/tulir/whatsmeow)'s `pair.go`. Drove **`xeddsa_sign`/`xeddsa_verify`** (Curve25519 XEdDSA signatures) into machin.

## Case study: a native WhatsApp client stack

The most demanding dogfood: building **every cryptographic and wire-format layer of WhatsApp natively in machin** ‚Äî a goal that looked impossible for a young language with no bignum, no crypto, and NUL-terminated strings. It's assembled and verified offline in **[machin-whatsapp](https://github.com/javimosch/machin-whatsapp)**, whose end-to-end self-test runs a message through the whole pipeline (`plaintext ‚Üí protobuf ‚Üí Signal ratchet ‚Üí WhatsApp stanza ‚Üí Noise transport ‚Üí` and back).

The five layers, each a standalone pure-MFL library pulled from the real protocol sources ([whatsmeow](https://github.com/tulir/whatsmeow), [libsignal](https://github.com/tulir/libsignal-protocol-go)):

| layer | tool | drove into machin |
|---|---|---|
| transport | [machin-noise](https://github.com/javimosch/machin-noise) ‚Äî Noise XX handshake | (composes the crypto builtins) |
| framing | [machin-wabin](https://github.com/javimosch/machin-wabin) ‚Äî WhatsApp stanza codec | (pure MFL over `bytes`) |
| body | [machin-protobuf](https://github.com/javimosch/machin-protobuf) ‚Äî protobuf codec | (pure MFL over `bytes`) |
| messages | [machin-signal](https://github.com/javimosch/machin-signal) ‚Äî Signal Double Ratchet | (libsignal-exact key schedule) |
| pairing | [machin-wapair](https://github.com/javimosch/machin-wapair) ‚Äî ADV pairing crypto | **`xeddsa_sign`/`xeddsa_verify`** |

Getting there grew the language itself: **`bytes`** (v0.34) ‚Üí the **crypto suite** (v0.35) ‚Üí **binary WebSocket** frames (v0.36) ‚Üí **declarable `bytes`** (v0.37) ‚Üí **XEdDSA** (v0.38). It stops at the live connection on purpose ‚Äî wiring an unofficial client to WhatsApp's servers risks an account ban ‚Äî so everything is verified offline by loopback/round-trip.

## In the machin repo

- **[`framework/`](https://github.com/javimosch/machin/tree/main/framework)** ‚Äî reusable MFL modules: `machweb` (a backend web framework ‚Äî router, response builders, JSON, binary bodies), `reactive` (a fine-grained signals + patch-list runtime for wasm frontends), and `flags.src` (a CLI flag parser), each composed ahead of your app with `machin encode`.
- **[`examples/`](https://github.com/javimosch/machin/tree/main/examples)** ‚Äî runnable programs: a raylib **GUI** game menu, JSON-over-HTTP servers, generics, concurrency, FFI, and more.

## Why these exist

machin's current direction is **dogfooding**: build real tools, hit real gaps, and grow the language to fill them. Each app above added capabilities to machin itself ‚Äî that's the point. To contribute an app, build something real and send a PR here with the link.

Several tools ship an agent **`SKILL.md`** at the repo root (e.g. [machin-meet](https://github.com/javimosch/machin-meet/blob/master/SKILL.md), [machin-notify](https://github.com/javimosch/machin-notify/blob/master/SKILL.md)) ‚Äî a focused, agent-facing guide to operating, integrating, and deploying the tool, including hard-won gotchas. Run `machin guide` to master the language itself.
