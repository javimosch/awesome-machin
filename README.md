# Awesome machin [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

A curated list of things built with **[machin](https://github.com/javimosch/machin)** тАФ MFL, the Machine-First Language. machin compiles a terse, type-inferred, agent-friendly language to native code through C: small static binaries, no runtime.

> New machin app? Open a PR adding a one-line entry. Keep the list machine-first: terse, linked, current.

## The language

- **[machin](https://github.com/javimosch/machin)** тАФ the compiler + language (MFL тЖТ C тЖТ native). Generics, closures, goroutines/channels (`select`, `close`, range, comma-ok), arena GC, C FFI, a `machweb` web framework, and a growing builtin surface (networking, native TLS + WebSocket, file I/O, SQLite, JSON + jq-style path queries, regex, base64, SHA-256/HMAC, `(value, err)` error handling).

> Organized by domain: **CLI tools & services**, **Game dev**, **Web**, **Crypto & protocols**. New repos follow a domain-scoped naming convention тАФ `machin-game-demo-*` for games/graphics, `machin-web-demo-*` for web.

## Learning & reference

- **[machin-learn](https://github.com/javimosch/machin-learn)** тАФ a **knowledge base for agents** learning MFL: seven skill files covering the language, builtins, gotchas/pitfalls, terminal/TUI, networking, storage/SQLite, and C FFI. Each skill is a self-contained `.md` an agent can load in one `read` call, with concrete examples and warnings for every caveat тАФ designed to help budget models master machin fast.

## CLI tools & services

- **[machin-terminal](https://github.com/javimosch/machin-terminal)** тАФ **[тЦ╢ landing](https://javimosch.github.io/machin-terminal/)** ┬╖ a **terminal emulator** in pure MFL (st-class, single-window): a raylib glyph grid driven by a hand-written **VT100/xterm parser** (alt-screen, scroll regions, SGR 16/256/truecolor, UTF-8 + box-drawing, **mouse reporting** X10/SGR) over a `forkpty` PTY. Hosts **vim, htop, and tmux** тАФ tmux is the multiplexer, so there are **no native tabs/splits by design**. One ~1.1 MB static binary, no GTK/VTE. Exercises the **PTY + `ioctl` + `poll` + raylib** C FFI surface тАФ the first non-game desktop GUI app in the ecosystem.
- **[machin-auth](https://github.com/javimosch/machin-auth)** тАФ **[тЦ╢ live demo](https://auth.intrane.fr)** ┬╖ a **self-hostable auth broker** (a self-hostable [Shoo](https://shoo.dev)): Google sign-in тЖТ a **domain-scoped, EdDSA-signed JWT** that any app verifies with a standard JOSE library against the published JWKS тАФ no SDK, no SaaS, no cross-app tracking (the subject is `HMAC(secret, google_sub | origin)`, so the same person is a different, unlinkable id at every app). One ~76 kB binary; composes `machweb` (signed cookies) + `sso.src` (the OAuth dance) + the crypto builtins (`ed25519_sign`/`ed25519_pub`, `hmac_sha256`, `base64`) тАФ **no new machin builtin**; the EdDSA JWT/JWKS are hand-built and verified against standard tooling.
- **[machin-playground](https://github.com/javimosch/machin-playground)** тАФ **[тЦ╢ try machin in your browser](https://play.intrane.fr)**. Write MFL, hit Run тАФ it compiles to **WebAssembly** and runs **client-side** (a Web Worker + a ~30-line WASI shim that captures `println`), so user code executes in the browser's wasm sandbox, never on the server. The whole thing is **one machin binary** (machweb) serving the editor page + a `/compile` endpoint (`machin build --target wasm`, bounded: тЙд24 KB, no FFI, timeouts). Building it drove a fix into machin тАФ wasm reactor modules now flush stdout ([machin#274](https://github.com/javimosch/machin/pull/274)).
- **[boilerplate-cli-ui-machin](https://github.com/javimosch/boilerplate-cli-ui-machin)** тАФ a single-binary CLI serving an embedded React web UI + JSON API, with a background daemon via the C FFI. The machin entry in the [SuperCLI](https://github.com/javimosch) boilerplate family (smallest binary at ~27 KB).
- **[machin-blob](https://github.com/javimosch/machin-blob)** тАФ a binary **blob inspector & patcher**: reads typed fields (`i32`, `f32`, `str`) at exact byte offsets via `peek_i32`/`peek_f32`, patches hex values at offsets, slices byte ranges. Drove **low-level memory ops** (`alloc`, `free`, `poke_u8`, `peek_i32`, `peek_f32`, `poke_i32`/`poke_f32`, `poke_ptr`) into the CLI-tool domain.
- **[machin-healthcheck](https://github.com/javimosch/machin-healthcheck)** тАФ a concurrent HTTP status/latency checker. One goroutine per URL, results over a channel. Drove `dial`/`now_ms`/`parse_int` into machin.
- **[machin-gen](https://github.com/javimosch/machin-gen)** тАФ a **template engine**: replaces `{{key}}` placeholders with values, supports transform pipelines (`{{name|upper|reverse}}`) via MFL's first-class **closures** stored in a `map[string]func` dispatch table. Transforms: upper, lower, reverse, trim, len, hex. Drove **closures** (`func` values, `[]func`, `map[string]func`) and **piped transforms** into the ecosystem.
- **[machin-hexdump](https://github.com/javimosch/machin-hexdump)** тАФ a binary **hex viewer**: classic hexdump format (offset / hex / ASCII sidebar), configurable width, colored output. Drove **`bytes`** builtins (`read_file_bytes`, `byte_at`, `to_hex`, `bytes_str`) and the `byte_to_hex` pattern into the CLI-tool domain.
- **[machin-ssg](https://github.com/javimosch/machin-ssg)** тАФ a static-site generator (markdown subset тЖТ HTML). Drove native file I/O and a parser fix into machin.
- **[machin-fetch](https://github.com/javimosch/machin-fetch)** тАФ a tiny HTTPS client CLI (a pocket `curl` for `https://`). GET/POST, single binary. Drove **native TLS** (`https_get`/`https_post`, OpenSSL) into machin.
- **[machin-wscat](https://github.com/javimosch/machin-wscat)** тАФ a tiny duplex WebSocket (`wss://`) client CLI (a pocket `websocat`). Stdin тЖТ send, recv тЖТ stdout. Drove a **native WebSocket client** (`wss_open`/`send`/`recv`/`close`, RFC 6455 over TLS) into machin.
- **[machin-linkcheck](https://github.com/javimosch/machin-linkcheck)** тАФ a concurrent link checker that classifies each URL (OK / 4xx / 5xx / unreachable) and exits non-zero on broken links. Drove **error handling** into machin: `http_get` returning `(status, body, err)` plus `exit`.
- **[machin-jq](https://github.com/javimosch/machin-jq)** тАФ a tiny `jq`-style JSON path query CLI (`.a.b[0].name`). Pairs with machin-fetch for a native HTTPSтЖТquery pipeline. Drove a **JSON path engine** (`json_get`) into machin.
- **[machin-pool](https://github.com/javimosch/machin-pool)** тАФ a bounded-concurrency URL checker: a fixed worker pool racing results against an overall deadline. Drove **`select`** (multi-way channel waits, timeouts) into machin.
- **[machin-pipe](https://github.com/javimosch/machin-pipe)** тАФ a streaming parallel fetch pipeline: producer тЖТ workers тЖТ fan-in, each stage terminating by closing its channel. Drove **channel `close`** + range-over-channel into machin.
- **[machin-batch](https://github.com/javimosch/machin-batch)** тАФ groups a line stream into JSON-array chunks by size or by timer (flushing the rest on EOF). Its `select` over input + ticker drove the **comma-ok receive** (`v, ok := <-ch`) into machin.
- **[machin-http](https://github.com/javimosch/machin-http)** тАФ a multi-command HTTPS client (`get`/`post`/`head`) with short+long flags and auto `--help`. Drove a reusable **flag parser** (`framework/flags.src`) into machin.
- **[machin-grep](https://github.com/javimosch/machin-grep)** тАФ a regex line matcher (pocket `grep`): match/invert/count/only/line-numbers over a POSIX ERE. Drove **regex** builtins (`regex_match`/`find`/`groups`/`replace`) into machin.
- **[machin-jwt](https://github.com/javimosch/machin-jwt)** тАФ a JWT decoder: decode header/payload (base64url JSON) and extract a claim. Drove **base64** builtins (`base64_encode`/`base64_decode`) into machin.
- **[machin-hmac](https://github.com/javimosch/machin-hmac)** тАФ compute/verify HMAC-SHA256 (or SHA-256) signatures тАФ webhook verification (GitHub/Stripe). Drove **hash** builtins (`sha256`/`hmac_sha256`) into machin.
- **[machin-wc](https://github.com/javimosch/machin-wc)** тАФ a `wc` clone (lines/words/bytes), matching coreutils. Drove **`read_stdin`** (slurp stdin verbatim) into machin.
- **[machin-kv](https://github.com/javimosch/machin-kv)** тАФ a persistent key-value store (set/get/del/list), data surviving across runs. Drove **SQLite** builtins (`sqlite_open`/`exec`/`query`/`close`) into machin.
- **[machin-cron](https://github.com/javimosch/machin-cron)** тАФ a cron-expression evaluator: prints the next N times a crontab line fires. Drove **`time_fields`** (decompose a Unix timestamp into calendar fields) into machin.
- **[machin-date](https://github.com/javimosch/machin-date)** тАФ a `date(1)`-style timestamp formatter (output matches GNU `date`). Drove **`time_format`** (strftime over a Unix timestamp) into machin.
- **[machin-mcp](https://github.com/javimosch/machin-mcp)** тАФ a **Model Context Protocol (MCP) server**: JSON-RPC 2.0 over stdio transport, exposing 6 tools (echo, now, read_file, write_file, list_dir, sqlite_query). The smallest MCP server in the ecosystem тАФ 31 KB static binary, no Node/Python runtime. Composes `json_get`/`json()` for JSON-RPC, `input()`/`println()`/`flush()` for stdio, `sqlite_open`/`sqlite_query` for databases, `read_file`/`write_file`/`list_dir` for filesystem тАФ no new builtins.
- **[machin-meet](https://github.com/javimosch/machin-meet)** тАФ a minimal self-hostable **Calendly for one person**: a single binary (HTTP + SQLite + embedded booking page) serving free slots, `.ics` files, and signed manage/cancel links, with booking alerts via **machin-notify** (Discord/Telegram) or WhatsApp. Drove **`time_make`** (build a Unix timestamp from calendar fields) into machin, completing the time trio.
- **[machin-qs](https://github.com/javimosch/machin-qs)** тАФ a URL query-string тЗД JSON converter (decode to a JSON object, or build a percent-encoded query from `key=value` pairs). Drove **`url_encode`/`url_decode`** (RFC 3986 percent-encoding) into machin.
- **[machin-notify](https://github.com/javimosch/machin-notify)** тАФ a notification **hub** (daemon + CLI): configure Discord webhooks / Telegram bots once, then apps `POST /notify {token, channel, message}` to one local endpoint тАФ no provider integration or secrets in each app. machin-meet sends booking alerts through it.
- **[machin-cms](https://github.com/javimosch/machin-cms)** тАФ an **agent-first headless CMS** (a CLI-based Strapi-lite) in one MFL binary: define collection schemas, CRUD data with typed validation, manage **RBAC** (users/roles/permissions), and expose chosen collections through a **REST API daemon** тАФ backed by SQLite, MySQL/MariaDB, or MongoDB (one env var picks the store). Agent-first I/O (JSON on stdout, structured errors on stderr, semantic exit codes 80тАУ119, `help-json`). Drove the **`system`** and **`sha1_bytes`** builtins (+ a pure-MFL **MySQL** driver) (run a shell command / spawn a detached daemon) into machin.
- **[machin-share](https://github.com/javimosch/machin-share)** тАФ a self-hostable **file & paste drop** (a private `transfer.sh`) in one MFL binary: drop a file (or paste text) and get a share link; list / download / inline-view / delete. Files on disk, metadata in embedded **SQLite**, no Node/object-store. Drove **file uploads** into machweb (v0.76.0): **binary-safe requests** (`req.body_bytes`), **`multipart/form-data` parsing** (`parse_multipart`/`multipart_file`/`multipart_field`), arbitrary response headers (`with_header`, for `Content-Disposition`), and the **`bytes_index`** / **`write_file_bytes`** / **`remove`** builtins. Verified a binary uploadтЖТdownload round-trips byte-for-byte (matching SHA-256).
- **[machin-live](https://github.com/javimosch/machin-live)** тАФ a self-hostable **live event & log stream hub** in one MFL binary: producers `POST /push/<topic>`, browsers/curl subscribe over **Server-Sent Events** (`GET /stream/<topic>`) and watch events arrive live; topic filtering, in-memory history replay, a live web console. Drove **streaming responses** into machweb (v0.77.0): `sse(fn)` + `sse_data`/`sse_event`/`sse_comment` (write a body incrementally over a long-lived socket) and **ignoring SIGPIPE** (a vanished client can't kill the server). Cross-goroutine fan-out via the idiomatic **lock-free hub goroutine** (a `chan` inside a struct, a `[]Sub` registry) тАФ verified for fan-out, filtering, history replay, and disconnect survival.
- **[machin-deploy](https://github.com/javimosch/machin-deploy)** тАФ a **production-ready reference machin web service**: one binary behind a reverse proxy (nginx/Caddy/Traefik/Cloudflare), wired correct + safe. Drove **reverse-proxy awareness + hardening** into machweb (v0.78.0): `scheme`/`client_ip`/`base_url` from `X-Forwarded-*` (`set_trust_proxy`), `Secure` cookies, a `413` body cap, read timeouts (anti slow-loris), JSON access logs, plus the **`peer_addr`** / **`socket_timeout`** builtins. Ships a systemd unit, a slim Dockerfile, and nginx/Caddy configs; new `machin guide --skill deploy`. Verified direct-vs-proxied scheme/IP, Secure cookies, `401`/`413`, and access logs end-to-end.
- **[machin-rooms](https://github.com/javimosch/machin-rooms)** тАФ a self-hostable **real-time chat server** in one MFL binary: multi-room WebSocket chat with broadcast fan-out, join/leave notices, and a live member count. Drove a **WebSocket *server*** into machweb (v0.79.0) тАФ the symmetric half of the wss client: the RFC 6455 upgrade handshake (`base64(sha1(key+GUID))`) + a **pure-MFL frame codec** over `bytes` + bitwise ops (`ws(req, fn)`, `ws_next_text`, `ws_send_text`, masked-client unmask), riding machweb's new generic **`hijack`** response (no new builtin). Lock-free hub-goroutine fan-out, two goroutines per connection; verified multi-client broadcast + room isolation against the Python `websockets` client.
- **[machin-mail](https://github.com/javimosch/machin-mail)** тАФ a self-contained **SMTP toolkit** in one MFL binary: **send** mail (an SMTP client) and **catch** it (a local SMTP sink + a web inbox, MailHog/Mailpit style), so the sender is testable with **zero external deps**. Drove an **SMTP toolkit** into machin (v0.80.0, `framework/smtp.src`): `smtp_send` (full `EHLO`/`AUTH LOGIN`/`MAIL`/`RCPT`/`DATA`/`QUIT`, base64 auth, multi-recipient, dot-stuffing) + `smtp_recv` (the server side of a session) + a buffered line reader тАФ pure MFL over `dial`/`read_bytes`/`base64`, no new builtin. Agent-first `send` CLI; verified both directions against Python stdlib `smtplib`.
- **[machin-hook](https://github.com/javimosch/machin-hook)** тАФ a self-hostable **webhook relay + live inspector** in one ~80 kB binary (webhook.site + smee.io, no Node/Docker/hosted dependency): capture any request to `/hook/<id>`, watch it land **live** in the browser (Server-Sent Events), inspect headers + body, **replay** it to any URL, or **forward** it to your local app. The first **flagship app** of the adoption push тАФ a thing a stranger would deploy because "one tiny binary on a $4 VPS" is the point. Composes `machweb` (HTTP + **`sse`**) + SQLite + the native HTTP client (`http_request`) for replay/forward; ~330 lines, captured payloads render via `textContent` (no XSS). **No new builtins** тАФ proof the language already composes into a real, deployable web app.
- **[machin-db-migrate](https://github.com/javimosch/machin-db-migrate)** тАФ a **SQLite тЗД MongoDB** migration CLI, both directions, in one pure-MFL binary (no driver, no `mongoexport`). Drove a pure-MFL **MongoDB client** into machin тАФ the **OP_MSG wire protocol** + a **BSON codec** ([`framework/mongo.src`](https://github.com/javimosch/machin/blob/main/framework/mongo.src) + [`bson.src`](https://github.com/javimosch/machin/blob/main/framework/bson.src)). `sqlite2mongo` reads the table's `PRAGMA` schema тЖТ typed BSON docs; `mongo2sqlite` infers columns from the first document тЖТ parameterized inserts; verified by a `SQLite тЖТ Mongo тЖТ SQLite` round-trip.
- **[machin-watch](https://github.com/javimosch/machin-watch)** тАФ a tiny **uptime monitor**: polls URLs on a per-watch interval and alerts on every upтЖФdown transition (with recovery time) through **machin-notify**. Composes `http_get` + SQLite + `sleep` тАФ no new builtins; closes the loop (deploy тЖТ notify тЖТ monitor).
- **[machin-pomo](https://github.com/javimosch/machin-pomo)** тАФ a terminal **Pomodoro timer**: live ANSI rendering, keyboard controls (pause/resume/skip/quit), configurable work/break durations, automatic long-break scheduling. Drove **`raw_mode`/`read_key`** (cbreak/no-echo + non-blocking single-key reads) into the CLI-tool domain: the terminal-input layer every future TUI inherits.
- **[machin-qr](https://github.com/javimosch/machin-qr)** тАФ a terminal **QR code generator**: encodes text into a scannable QR code via MFL's C **FFI** (`libqrencode`), rendered with ANSI background colors. Drove **`extern`** (FFI declarations), **raw memory access** (`peek_i32` for struct fields), and **pointer reconstruction** (64-bit little-endian from two i32 halves) into the ecosystem.
- **[machin-revproxy](https://github.com/javimosch/machin-revproxy)** тАФ a **reverse proxy**: route HTTP requests to backend servers by path prefix, with header rewriting (Host, X-Forwarded-*) and prefix stripping. Longest-prefix match routing, concurrent goroutines per request. Composes `dial`/`read`/`write` for TCP relay, `read_bytes` for binary-safe response forwarding, `has_prefix` for route matching тАФ no new builtins.
- **[machin-scan](https://github.com/javimosch/machin-scan)** тАФ a concurrent **port scanner**: goroutine worker pool, bounded concurrency, channel-based result collection via `select`, optional timeout. Scans port ranges, comma-separated lists, or single ports with service name lookup (30+ common ports). Drove **channels** (`make(chan T)`/`<-ch`/`close(ch)`) and **`select`** (multi-way channel wait) into the CLI-tool domain тАФ MFL's full concurrency model.
- **[machin-serve](https://github.com/javimosch/machin-serve)** тАФ a static file **HTTP server**: concurrent (goroutines per request), opt-in Basic auth, MIME-type detection, directory index (`index.html`), path-traversal protection. Binds to `0.0.0.0` with configurable port and directory. Composes `listen`/`accept`/`read`/`write`/`close` + `read_file` + `regex_groups` + `base64_decode` + `go` тАФ no new builtins.
- **[machin-top](https://github.com/javimosch/machin-top)** тАФ a live **system dashboard**: real-time CPU, memory, uptime, and load average with ANSI-colored progress bars. Reads `/proc/stat`, `/proc/meminfo`, `/proc/uptime`, `/proc/loadavg` via `read_file`. Refreshes live with `sleep` and ANSI cursor control from pomo тАФ no new builtins.
- **[machin-vault](https://github.com/javimosch/machin-vault)** тАФ an **agent-first backup vault** for files & databases in one self-hostable binary: `backup`/`list`/`get`/`unpack`/`restore` over a versioned-JSON contract (semantic exit codes 80тАУ119, `help-json`, `--yes` gates on destructive ops) so an agent can recover a lost database **at a glance** тАФ plus an **isomorphic dashboard** the same binary serves (SSR HTML that works with no JS, progressively enhanced by a **wasm** client compiled from the same MFL). Backs up `file`/`postgres`/`mysql`/`mongo`/`custom` targets by orchestrating the real tools (`rsync`, `pg_dump`, `mongodump`, тАж) тАФ content-addressed **dedup**, per-target **retention**, optional **aes-256 encryption** at rest; stateless storage (`list` just scans disk). Drove the **`exec(cmd) -> (exit_code, stdout, stderr)`** builtin into machin (v0.83.0) тАФ capture a subprocess's stdout/stderr (the gap `system` left), unblocking SSH / `mongodump` / `gzip` pipelines.
- **[machin-open-serpapi](https://github.com/javimosch/machin-open-serpapi)** тАФ **[тЦ╢ landing](https://javimosch.github.io/machin-open-serpapi/)** ┬╖ an **agent-first, self-hostable SerpApi alternative**: one MFL binary that parses rendered search-engine HTML into clean JSON тАФ **Google** organic + **Google Maps** places (hand-written structure-anchored extractors), and **DuckDuckGo**, **Bing**, **Startpage** + **Brave** defined as **pure-JSON selector specs** run by a tiny built-in **CSS-selector engine** (HTMLтЖТflat DOMтЖТ`tag`/`.class`/`#id`/`[attr]` with descendant + child combinators, right-to-left matching). Owns SerpApi's **parse + schema** layers and makes **fetch a plugin** тАФ any program that prints a page to stdout (curl, a cached fixture, or an **anti-detect browser** fetcher that gets past the Google/DDG/Bing CAPTCHA walls) тАФ so the binary never touches the network and is **fully offline-testable** against frozen real-page fixtures. Composes `read_stdin`/`json_get`/`read_file`/`base64_decode`/`url_decode` тАФ **no new builtin**; adding a clean-markup engine is a JSON file, not code.

- **[machin-game-demo-flappy](https://github.com/javimosch/machin-game-demo-flappy)** тАФ a **Flappy-Bird** with real PNG **sprites**: `Texture2D` handles returned across the FFI, `f32`-field `Rectangle`/`Vector2` structs for `DrawTextureRec`/`DrawTexturePro`, a sprite sheet rotated by velocity, pipes flipped via a negative source height. ## Web

> machin does both ends of the web in one language тАФ a native server (machweb) and a reactive WebAssembly UI тАФ no Node/bundler. The agent how-to is the [`machin-web` skill](https://github.com/javimosch/machin/blob/main/skills/machin-web/SKILL.md); the direction is the [web north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-WEB.md). (Web *services* like [machin-meet](https://github.com/javimosch/machin-meet) and [machin-ssg](https://github.com/javimosch/machin-ssg) are listed under CLI tools & services.)

- **[machin-saas-demo](https://github.com/javimosch/machin-saas-demo)** тАФ **MachNotes**, an end-to-end **SaaS in one binary**: the whole backend stack wired together тАФ **OAuth2/OIDC SSO** login ([`sso.src`](https://github.com/javimosch/machin/blob/main/framework/sso.src), with a bundled mock identity provider so it runs locally), **sessions in Redis** (session id тЖТ email, TTL; [`redis.src`](https://github.com/javimosch/machin/blob/main/framework/redis.src)), a **parameterized Postgres** notes table ([`postgres.src`](https://github.com/javimosch/machin/blob/main/framework/postgres.src)), signed cookies + routing ([`machweb.src`](https://github.com/javimosch/machin/blob/main/framework/machweb.src)), and a **reactive WebAssembly UI** ([`reactive.src`](https://github.com/javimosch/machin/blob/main/framework/reactive.src) тЖТ wasm, keyed-list patches). Login тЖТ store note тЖТ reactive list тЖТ logout, verified end to end. The backend capstone тАФ proof a single static MFL binary is a credible SME backend (HTTP + auth + SQL + cache), no Node/cgo/client-libs. Concurrency-safe via connection pooling (the gap it surfaced, added in machin v0.65.0). See the [backend north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-BACKEND.md).
- **[boilerplate-cli-ui-machin-isomorphic](https://github.com/javimosch/boilerplate-cli-ui-machin-isomorphic)** тАФ a **starting point for isomorphic web apps in machin**: one static native binary that is a CLI, an HTTP server, a JSON API, and serves its own **reactive WebAssembly UI** тАФ the whole app (server, client, shared model) in MFL, no Node/bundler. The example poll is **server-rendered** (works with JS off), then **hydrated** by a wasm client where signals + a computed total + templating drive surgical DOM updates; `models.src` is compiled into *both* ends. Composes everything: `machweb` + `flags` + `reactive` + `--target wasm` + binary-body serving. The full-stack-MFL capstone.
- **[machin-web-demo-router](https://github.com/javimosch/machin-web-demo-router)** тАФ a **multi-page SPA** with client-side routing: three pages, no-reload navigation, working back/forward + address bar, and deep-links (refresh on `/users`). The router ([`framework/router.src`](https://github.com/javimosch/machin/tree/main/framework), on top of reactive) makes the active route a **signal** (an int index); an `outlet` reaction re-renders the active page, `link`/`navigate` switch routes, and path strings flow hostтЖТwasm via `ptr_str`. One page is reactive тАФ a counter whose state **persists across navigation** (it's a signal). A catch-all machweb server serves the shell for any path. All MFL.
- **[machin-web-demo-users](https://github.com/javimosch/machin-web-demo-users)** тАФ a **CRUD back-office** (manage a users database): one native binary that is a CLI + HTTP server + **SQLite**-backed JSON API + reactive wasm UI (list / add / toggle active / delete). The server owns the data (`sqlite_query` тЖТ JSON rows, `?`-bound params); the client is a reactive *view* тАФ the host fetches `/api/users` and hands the JSON to the wasm (`ptr_str` + `json_get`), a keyed list patches only changed rows, a computed shows the active count, and a form adds users. The SPA-over-API counterpart to the isomorphic boilerplate; the worked "manage my users DB" recipe.
- **[machin-web-demo-todo](https://github.com/javimosch/machin-web-demo-todo)** тАФ a reactive **todo app with real text input**: state, logic, and rendering all in machin, compiled to wasm. The new piece is a working `<input>` тАФ the typed UTF-8 flows *into* the module (machin v0.57's **`ptr_str`**: the host writes the text into wasm memory at an `alloc`'d pointer, machin reads it). Signals + a computed "active" count + a **keyed list** (re-keyed by `id*100+done` so a toggle re-renders just that row) + templating. Adding inserts one row, toggling re-renders one, deleting removes one тАФ no `innerHTML` churn. Rounds out the toolkit with **forms**.
- **[machin-web-demo-reactive](https://github.com/javimosch/machin-web-demo-reactive)** тАФ **fine-grained reactivity** (the Solid/Leptos model, in MFL): a reactive list whose **state and view logic live entirely in machin** тАФ **signals**, a **computed** sum, live bindings, and a **keyed list** that emits only the DOM deltas (`insert`/`remove`/`reorder`), no `innerHTML` replacement, no virtual-DOM diff. The runtime ([`framework/reactive.src`](https://github.com/javimosch/machin/tree/main/framework)) auto-tracks which reactions read which signals, recomputes only the affected ones, and patches only when the rendered text/keys actually change тАФ sorting the list emits *only* a reorder, no item re-render. Drove **`[]func`** (slices of closures) into machin (v0.53.0) and **computed + keyed reconciliation** (v0.54.0); composes package globals (the signal store) + `--target wasm`.
- **[machin-web-demo-ssr](https://github.com/javimosch/machin-web-demo-ssr)** тАФ **isomorphic server-side rendering**: a counter page rendered by a single native machin binary (machweb), where the view logic lives in a shared `view.src` that *also* compiles to the wasm client тАФ **one component model, two runtimes**. The server's HTML and the wasm client's re-render are byte-for-byte identical (same `view()`). SSR needs **no JavaScript** (the +/тИТ controls are plain server-round-trip links); the wasm client adds an instant SPA from the same source. No Node, no bundler тАФ `machin encode` + `machin build`. The full-stack-MFL direction: write a component once, run it on the server for first paint and in wasm for interactivity.
- **[machin-web-demo-wasm](https://github.com/javimosch/machin-web-demo-wasm)** тАФ **machin on the frontend**: an interactive counter whose entire view (markup, `n┬▓`, a recursive `fib`) is written in MFL, compiled to **WebAssembly**, and rendered live in the browser тАФ the way Rust does with Yew/Leptos, here **MFL тЖТ C тЖТ wasm** reusing machin's C backend. `zig cc` (clang + wasi-libc, one binary) cross-compiles the emitted C to a `wasm32-wasi` reactor module with zero WASI imports, so it loads in a bare page. The DOM bridge **is machin's own FFI, both ways**: exported MFL functions are wasm exports JS calls into; an `extern "env"` block becomes a wasm import the host supplies (machin тЖТ `innerHTML`). Composition тАФ machin reaches the browser with no new language primitive, only a leaner emit. The first **frontend** dogfood; see the [web north star](https://github.com/javimosch/machin/blob/main/docs/NORTH-STAR-WEB.md).
## Crypto & protocols

> The pure-MFL layers behind the native WhatsApp client stack (see the case study below).

- **[machin-protobuf](https://github.com/javimosch/machin-protobuf)** тАФ a protobuf wire-format inspector (`protoc --decode_raw`-style) + reusable codec library, implemented in pure MFL over the **`bytes`** type. Exercises machin's binary surface (bytes + crypto + binary WebSocket) built out for a native WhatsApp client.
- **[machin-noise](https://github.com/javimosch/machin-noise)** тАФ a [Noise Protocol](https://noiseprotocol.org) **XX handshake** (`Noise_XX_25519_AESGCM_SHA256`, the pattern WhatsApp uses) in pure MFL over the `bytes` + crypto builtins, with a loopback self-test. The transport-encryption layer for a native WhatsApp client.
- **[machin-wabin](https://github.com/javimosch/machin-wabin)** тАФ the **WhatsApp binary-node (stanza) codec** in pure MFL: token-dictionary compression (from [whatsmeow](https://github.com/tulir/whatsmeow)), nibble/hex packing, the node tree. Encoder output is byte-for-byte WhatsApp-compatible. The stanza-framing layer for a native WhatsApp client.
- **[machin-signal](https://github.com/javimosch/machin-signal)** тАФ the **Signal Double Ratchet** (X3DH + ratcheting cipher) in pure MFL, key schedule matching [libsignal](https://github.com/tulir/libsignal-protocol-go) exactly, with a loopback self-test. The message-encryption layer for a native WhatsApp client.
- **[machin-wapair](https://github.com/javimosch/machin-wapair)** тАФ WhatsApp **device-pairing crypto** (the ADV account/device signature exchange) in machin, modeling [whatsmeow](https://github.com/tulir/whatsmeow)'s `pair.go`. Drove **`xeddsa_sign`/`xeddsa_verify`** (Curve25519 XEdDSA signatures) into machin.

## Case study: a native WhatsApp client stack

The most demanding dogfood: building **every cryptographic and wire-format layer of WhatsApp natively in machin** тАФ a goal that looked impossible for a young language with no bignum, no crypto, and NUL-terminated strings. It's assembled and verified offline in **[machin-whatsapp](https://github.com/javimosch/machin-whatsapp)**, whose end-to-end self-test runs a message through the whole pipeline (`plaintext тЖТ protobuf тЖТ Signal ratchet тЖТ WhatsApp stanza тЖТ Noise transport тЖТ` and back).

The five layers, each a standalone pure-MFL library pulled from the real protocol sources ([whatsmeow](https://github.com/tulir/whatsmeow), [libsignal](https://github.com/tulir/libsignal-protocol-go)):

| layer | tool | drove into machin |
|---|---|---|
| transport | [machin-noise](https://github.com/javimosch/machin-noise) тАФ Noise XX handshake | (composes the crypto builtins) |
| framing | [machin-wabin](https://github.com/javimosch/machin-wabin) тАФ WhatsApp stanza codec | (pure MFL over `bytes`) |
| body | [machin-protobuf](https://github.com/javimosch/machin-protobuf) тАФ protobuf codec | (pure MFL over `bytes`) |
| messages | [machin-signal](https://github.com/javimosch/machin-signal) тАФ Signal Double Ratchet | (libsignal-exact key schedule) |
| pairing | [machin-wapair](https://github.com/javimosch/machin-wapair) тАФ ADV pairing crypto | **`xeddsa_sign`/`xeddsa_verify`** |

Getting there grew the language itself: **`bytes`** (v0.34) тЖТ the **crypto suite** (v0.35) тЖТ **binary WebSocket** frames (v0.36) тЖТ **declarable `bytes`** (v0.37) тЖТ **XEdDSA** (v0.38). It stops at the live connection on purpose тАФ wiring an unofficial client to WhatsApp's servers risks an account ban тАФ so everything is verified offline by loopback/round-trip.

## Why these exist

machin's current direction is **dogfooding**: build real tools, hit real gaps, and grow the language to fill them. Each app above added capabilities to machin itself тАФ that's the point. To contribute an app, build something real and send a PR here with the link.

Several tools ship an agent **`SKILL.md`** at the repo root (e.g. [machin-meet](https://github.com/javimosch/machin-meet/blob/master/SKILL.md), [machin-notify](https://github.com/javimosch/machin-notify/blob/master/SKILL.md)) тАФ a focused, agent-facing guide to operating, integrating, and deploying the tool, including hard-won gotchas. Run `machin guide` to master the language itself.
